## 日程管理功能产品设计 + 实现计划
- 输出日程管理功能产品设计方案，核心定位为「孩子的通用个人日历」。
- 日程项分两类：关联任务型（taskId 非空，到时间可去打卡）和独立事件型（taskId 为空，仅提醒）。
- 入口方案：主页底部三按钮（兑换时间 / 查看记录 / 我的日程），日程页含今日卡片 + 周计划总览 + 全部计划列表 + 新建/编辑弹窗。
- 数据存储：`data/schedules/<child-id>.json`，原子写入（tmp → rename），孩子通过界面管理。
- 提醒策略：第一期做应用内提示 + 前台定时提醒；推送通知作为第二期（需 HTTPS）。
- 用户确认设计方向后，输出完整实现计划 `timebank-schedule-plan.md`，含数据模型、API 设计、组件清单、测试计划、5 个里程碑（S1-S5）。

## 日程功能大幅调整（解耦+表格+schema 变更）
- **Schema 变更**：`time` → `startTime` + `endTime`（均必填，endTime > startTime）；移除 `taskId`（日程与任务完全解耦）；颜色枚举改为 learn/play/chore/other。
- **Zod refine + extend 冲突**：`.refine()` 返回 ZodEffects 不支持 `.extend()`，改为先定义 base 对象再分别 refine。
- **WeekCalendarGrid**：隐藏滚动条（scrollbarWidth:none + ::-webkit-scrollbar{display:none}）解决表头比单元格宽的问题；移除全天区；日程项改为可点击 button 触发编辑。
- **SchedulePage**：移除"全部计划"列表；今日计划移至底部；点击日历格子和今日项均可编辑。
- **ScheduleEditor**：移除关联任务和不设时间选项；加 endTime 输入；颜色改为学习/娱乐/家务/其它。
- **HomePage 提示条**：改用 `!isOverdue` 统计未结束的今日日程。
- **server today 接口**：移除 taskName/taskCompleted 逻辑。
